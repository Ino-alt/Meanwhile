# Meanwhile

## 1. サービス概要

Meanwhileは、"ランダムな国の「今（現地時刻＋短い状況文）」を表示し、その隣に自分の「今」"を短く書き残せる日記アプリです。
仕事や人間関係で視野が狭くなりやすい人が、世界の時間と並べて自分の現在地を見直すきっかけを作ります。

### 誰の、どんな課題を、どう解決するか

- 誰：生活圏（職場・家族・近い人間関係）だけが世界の全てに感じて苦しくなる人
- 課題：思考が近い範囲に閉じてしまい、「今の自分」しか見えなくなること
- 解決：世界の「今」を先に提示し、「あなたの今」を書かせる構図で、視点の固定を緩める

---

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情

仕事が立て込み、疲労が溜まっていた時期がありました。
毎日同じ時間に起き、同じ席に座り、同じ人間関係の中で過ごす日々。

- 職場での評価が気になる
- 頭の中がその範囲だけで埋まる
- 日々が「コピペ」のように感じる
- 時間が流れている実感が薄くなる

「ここで嫌われたら終わり」「この生活圏が世界の全てだ」という感覚になり、息苦しさを覚えました。

### 2-2. なぜそれが「気になった」のか

- 視野が狭くなると同じ考えを繰り返してしまう
- 「今いる場所」だけが世界の全てだと思い込む
- 判断や感情が極端になる

励ましではなく、"世界の別の時間を隣に置く仕組み"が必要だと考えました。

---

## 3. 課題の整理

### 表に見えている困りごと

- 仕事や人間関係で悩むと、視野が生活圏だけに閉じてしまう
- 毎日が同じに感じられ、時間が流れている実感が薄くなる
- 「今ここ」が全てに感じて、気持ちが追い込まれやすい

### 本当に解決したい課題は何か

- なぜ困る？→ 視野が狭くなると、目の前の問題を過大評価してしまう
- なぜ過大評価してしまう？→ 他の時間軸や場所を想像できず、「ここが全て」だと錯覚するから
- なぜそれが問題？→思考が固定され、感情が極端になり、自分を追い込んでしまう

### 本質的な課題

自分の時間と世界の広がりを実感できず、思考と感情が生活圏の内側に固定されてしまう状態を、
一度緩めるきっかけがないこと。

## 4. 想定ユーザーについて

### 4-1. 想定しているユーザー

#### 年齢・ライフステージ

- 20代後半〜30代
- 仕事や人間関係の比重が高くなりやすい時期

#### 状況

- 日々の変化が少ない

#### 環境

- 職場や家庭など逃げ場が少ない固定化されたコミュニティ

### 4-2. 自分とユーザーの距離

- 自分自身
　　→ 自分が同じ状態になりやすく、実感に基づいて設計できるため

### 4-3. 実際に使われる可能性について(需要・存在確認)

- 最初に使ってくれそうな人：自分、または近い友人
- 思い出す場面：考えが狭くなっている時、夜に疲れている時、気持ちが詰まっている時
- 代わりに使っているもの：メモアプリ、日記アプリ、SNSの下書き
- 置き換える理由：メモや日記より入力が短く、SNSより安全で、世界の「今」がトリガーになる

### 4-4. 導入・継続の流れ

- きっかけ：気持ちが詰まった時 / 思考が同じ場所を回っている時に「一旦切り替えたい」と思う
- 最初の一歩：アプリを開く → 世界の「今」が表示される → 「あなたの今」を1行で入力する
- 継続理由：入力が短く負担が少ない／書くことで思考の渋滞が一度ほどける（“その場”の区切り）

## 5. 既存サービス・競合調査

### 5-1. 似たサービスの調査

- サービス名：引き出し日記
- できること：短文を日記として記録し、タグなどで管理できる

### 5-2. それでも自分が作りたい理由

- 既存の日記アプリは「自分の内側」だけで完結しがちだと感じました
- いま自分が欲しいのは、記録機能そのものよりも、“視点をずらす導線”です
- Meanwhile は「世界の今」を先に置くことで、書く前の思考の向き（内向き固定）を少し動かすことを狙っています

### 5-3. 差別化の要点

このアプリの差別化ポイントは、世界の「今」を先に提示し、「あなたの今」を並べて書かせる構図です。

## 6. このサービスで提供したい価値

### 6-1. ユーザーの変化

- 行動の変化：気持ちが詰まった時に、短い入力で“区切り”を作られる
- 気持ちの変化：思考が一点に集中している状態から少し離れられる
- 考え方の変化：「ここだけが世界ではない」を、説教ではなく体験として思い出せる

### 6-2. 価値を一文で表す

Meanwhile は、視野が生活圏に固定されがちな人が、世界の今と並べて自分の今を書き、思考の詰まりを緩められるサービスです。

## 7. このアプリで実現すること

### MVPで作る機能
　　
1. 起動時/更新時に「ランダム国（５~10カ国を想定）」を選び、現地の現在時刻を表示
2. 現地時刻から 時間帯（朝/昼/夕/夜） を判定し、時間帯に対応した 定型文をランダム表示
	    例：「現在、イスタンブールは12:12です。昼食をとっている頃です。」（※文は複数候補からランダム）
3. 質問（例：「いま、あなたはどんな時間を過ごしていますか？」）を表示
4. 投稿機能（複数投稿可）
5. 「更新」機能：ボタン押下で世界スナップショットを再生成。
	　　　- ランダム国を再選択
	　　　- 現地時刻を再計算
	　　　- 時間帯判定と定型文を再抽選
         - その結果を新しいセッションとして保存
6. 世界情報（国・時刻・定型文）と投稿をセットで保存
7. セッション一覧

### MVPの利用条件（制約）

- MVPでは認証機能（ログイン）は実装しない
- ユーザーはブラウザ単位のゲストユーザーとして識別する（guest_idを利用）
- 本リリースで認証機能を追加予定

### 本リリースで作る機能

- カレンダー表示（日に紐づくセッションの整理）
- 国や定型文のパターンの追加
- 自分の現在地（ローカル時刻）の併記
- 認証機能（ログイン）を追加し、データの永続化・機種変更・複数端末利用に対応

## 8. このアプリの懸念点とその対策

### 懸念点：日記アプリとしては差が弱く、継続理由が薄くなる可能性

- 何が問題になりそうか：ただの短文メモになって、続かない
- なぜそう思うか：継続のフックが「記録したい人」以外に届かない可能性

### 対策

- “起動時に世界の今が出る”体験を核としてぶらさない
- セッション一覧で「世界×自分」のセットが見返せる形にする
- MVP後にカレンダー等で振り返り体験を強化する

---

## 9. 今後の展開・発展の方向性

- ユーザーはなぜ使い続ける？：入力が短く、負担が小さい。溜まると振り返りが効く
- 続けることで何が変わる？：自分が詰まりやすいタイミング/状況が可視化される
- データが溜まると何ができる？：日付別、国別での振り返り、傾向の発見（将来的に）

### 今考えている発展案

- 現地時刻の自動更新（分単位更新、秒なし表示など）
- フィルタ/検索（国・日付）
- 任意の国を選択し、その国の時間と1日を並行して過ごせる「固定国モード」
- 機能追加の方向性：カレンダー、国・状況文の増加、自動更新
- 機能以外の工夫：問いかけ文の精度を上げる（行動だけでなく“過ごし方”に寄せる）

## 10. 技術スタック

### 10-1. 使用予定の技術

- フレームワーク：Ruby on Rails
- デプロイ先：Render
- データベース：PostgreSQL
- CSS フレームワーク：Bootstrap（または Tailwind CSS）

### 10-2. 技術選定の理由

- なぜこの技術を使うのか：学習カリキュラムに沿っており、MVPを確実に作成できるため
- 今回チャレンジしたい点：「世界スナップショット（国・時刻・定型文）」と投稿をセッションとして紐づける設計
- 不安な点：タイムゾーンの扱いと、現地時刻・時間帯判定の正確性
- 対応：国ごとのタイムゾーンをテーブル（または定数）で管理し、時刻変換と時間帯判定を最小構成で先に検証する（例：数カ国で表示確認）
